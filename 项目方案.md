# 抖音数据爬取与情感分析系统 — 第一阶段方案

## Context

科研用途的抖音数据采集与情感分析系统。不依赖官方 API，通过爬虫抓取视频文字描述、评论及其元数据（发布者、时间、IP 属地等），然后进行情感分析、观点挖掘。

**第一阶段目标**：实现最小可行工程 —— 爬虫 + 情感分析。预警系统和前端留到后续进阶阶段。

## 技术选型

| 层次 | 技术 | 理由 |
|---|---|---|
| 语言 | Python 3.12 | 爬虫+NLP 生态最佳 |
| 爬虫 | Playwright + playwright-stealth | 反检测能力强，优于 Selenium |
| 数据库 | MongoDB 7.0 | 灵活 schema，适合非结构化爬取数据 |
| 中文分词 | jieba | 成熟、CPU 友好 |
| 情感分析 | bert-base-chinese + LoRA 微调 | GPU 推理，高精度中文情感分类 |
| 情感分析 (fallback) | PaddleNLP Senta-BiLSTM | CPU 可跑，备用方案 |
| 观点抽取 | PaddleNLP UIE | 通用信息抽取，适配中文 |

## 反爬策略

- `playwright-stealth` 自动规避 `navigator.webdriver` 等检测
- 高斯分布随机延迟（均值 2s，标准差 0.8s）
- 人类行为模拟：随机鼠标移动、变速滚动、随机标签切换
- Session 管理：每 20-30 请求轮换浏览器上下文
- 硬性限制：每视频页 3-5 秒间隔，每天 1000 次请求上限
- 不做移动端 API 绕过（后续阶段可选）

## 数据流

```
[run_scraper.py]
    │
    ▼
[crawler.platform] ──▶ Playwright 浏览器 ──▶ douyin.com
    │
    ▼ (upsert)
[MongoDB: videos, comments]
    │
    ▼ (读取未分析数据)
[run_analysis.py]
    │
    ▼
[预处理: cleaner → normalizer → tokenizer]
    │
    ▼
[情感分析: BERT / PaddleNLP] ──▶ [观点抽取: UIE]
    │
    ▼ (写入结果)
[MongoDB: sentiment_results]
    │
    ▼ (输出)
[CSV/JSON 导出 + 统计摘要打印到控制台]
```

## 实施步骤

### Step 0 — 环境搭建
- Docker 启动 MongoDB
- 创建 Python 虚拟环境，安装依赖
- 安装 Playwright 浏览器
- 预下载 NLP 模型

### Step 1 — 存储层
- 配置系统（YAML + 环境变量）
- MongoDB 连接管理 + 索引创建
- Pydantic 数据模型 + CRUD 封装

### Step 2 — 核心爬虫
- Playwright 浏览器生命周期 + 反检测
- 限速器 + 重试机制
- 视频页解析器 -> 评论解析器 -> 用户解析器
- 搜索 + 视频列表 + 评论滚动加载
- `run_scraper.py` 命令行入口

### Step 3 — NLP 分析管线
- 预处理：文本清洗 + jieba 分词 + 繁转简 + 网络用语词典
- 人工标注 200-500 条评论
- LoRA 微调 bert-base-chinese
- 情感分类（BERT 主力 + PaddleNLP fallback）
- 观点方面抽取（PaddleNLP UIE）
- `run_analysis.py` 命令行入口：读取未分析数据 → 分析 → 写入结果 → 输出统计摘要

## 已确认的约束

| 维度 | 选择 |
|---|---|
| 数据规模 | 小规模（每次几十~几百条视频） |
| 弹幕 | 暂不需要 |
| GPU | 有 GPU，用 BERT + LoRA |
| 实时性 | 批量分析，命令行触发 |
| 前端 | **第一阶段不做** |
| 预警 | **第一阶段不做** |
| API | **第一阶段不做** |

## 验证方案

1. 成功爬取 10 个指定关键词的视频元数据 + 评论，写入 MongoDB
2. 对爬取评论做情感分析，人工抽查 50 条验证准确率
3. 导出分析结果为 CSV，确认数据结构完整
